# 强化学习与量化投资：数据获取与标的选取说明

本文档详细记录了在深度强化学习投资组合项目（EDP 等算法研究）中，关于“投资标的选取”及“历史行情数据获取”的完整策略与实现流程。本流程的设计充分考虑了量化投资的实盘可执行性以及模型训练的稳定性。

---

## 1. 标的选取策略：为何选取“沪深 300 权重前 20 股”

在强化学习算法回测与实盘中，股票池的选取极其重要，直接影响算法的有效性和落地可能。我们最终**放弃了随机抽取，选择了沪深 300 官方权重排名前 20 的成分股**。

### 1.1 随机抽取的弊端（为何不采用）
- **流动性枯竭风险**：沪深 300 虽整体代表性强，但尾部股票的流动性相对较弱。如果随机抽取，可能选中小市值、低流动性的股票，导致模型调仓时遇到“有价无市”的情况，产生极大的滑点（Slippage）和冲击成本（Impact Cost）。
- **停牌与黑天鹅事件**：随机抽取容易引入存在基本面瑕疵的个股，这些个股受新闻或突发事件（如停牌、ST 等）影响较大。此类噪音数据不利于深度强化学习模型从价格中学习到稳定的特征表征。

### 1.2 选取 Top 20 权重股的核心优势
1. **强代表性与基准对齐**：沪深 300 指数中，前 20 大权重股占据了整个指数**约 35% 的权重**，它们是指数涨跌的核心驱动力。这也方便我们后续将“买入并持有（Buy & Hold）沪深300”作为基准（Baseline）进行公平对比。
2. **极高的流动性与容量**：前 20 大成分股（如贵州茅台、宁德时代、中国平安等）是 A 股市场流动性最好的资产，单日成交额通常在几十亿甚至上百亿级别。强化学习频繁的调仓操作在这些标的上的冲击成本可以忽略不计。
3. **模型收敛的稳定性**：大盘蓝筹白马股的走势受宏观经济和行业周期驱动更明显，具备更好的统计学规律，信噪比高，有助于深度网络快速捕捉市场趋势特征并稳定收敛。
4. **文献复现标准**：学术界主流的量化 RL 论文（如应用 DDPG、PPO 于资产组合管理）在 A 股或美股的测试集上，几乎全部采用类似 “Top 30” 或 “Top 50” 的高流动性股票组合。

## 2. 数据获取流程与实现方案

整个数据获取流程分为**两步**：第一步是通过官方中证指数接口获取最新的成分股权重排名并提取前 20 名；第二步是针对这 20 只股票拉取长期的日线级别历史行情。

我们基于开源金融数据接口库 **AkShare** 进行实现。由于复杂的网络环境（如开发环境中可能存在的代理拦截），脚本在设计时加入了自动降级（Fallback）、重试与断点续传机制，确保数据顺利落盘。

### 2.1 步骤一：获取官方权重 Top 20

- **数据源**：中证指数官方（经 AkShare `index_stock_cons_weight_csindex` 接口封装）。
- **逻辑**：请求沪深 300（000300）的最新成分股权重量表，按权重降序排列，取前 20 名。
- **输出**：生成一份映射清单文件 `top20_hs300_by_official_weight.csv`。包含如下核心列：
  - `symbol`: 股票代码（如 600519）
  - `name`: 股票名称（如 贵州茅台）
  - `weight`: 最新官方权重占比（如 3.737%）

### 2.2 步骤二：拉取近三年历史 K 线数据

在获取到 Top 20 股票代码后，我们需要获取其历史行情用于 RL 训练（包含状态特征和收益率计算）。

- **时间跨度**：动态获取，从代码执行当日起，**倒推 3 年（即 365×3 天）**。
- **数据频率与处理**：日线级别（Daily），并使用**前复权（qfq）**处理，消除分红送配导致的除权缺口，保证价格连续性，这是计算资产收益率的硬性要求。
- **数据源选择与降级策略**：
  - AkShare 默认拉取日线常采用东方财富接口，但在开启科学代理的系统环境中，东财接口（`push2his.eastmoney.com`）极易报 `RemoteDisconnected` 或 `ProxyError`。
  - 为此，我们在最终使用的脚本中强制切换至 **新浪财经数据源 (`stock_zh_a_daily`)**。新浪数据源对本地网络代理的环境兼容性较好。
- **稳健性保障（Robustness）**：
  - **断点续传**：遍历名单时，若检测到该股票的 CSV 已经存在且非空，则跳过拉取，直接进行下一只。
  - **失败重试**：单次 HTTP 请求如果遭遇超时或报错，会执行 3 次重试，每次重试间隔 2 秒。
  - **频率限制**：成功拉取一只后自动休眠 1 秒（Sleep），防止触发 API 提供方的封控限流机制。

## 3. 获取的数据格式说明

最终拉取完成的数据统一存放在 `data_thesis/top20_weight_3years_data/` 目录下，每只股票独立生成一个 CSV（例如 `600519_贵州茅台.csv`），可以直接作为强化学习中 `TradingEnv` 或 `PortfolioEnv` 的初始化加载数据。

单只股票的 CSV 数据包含以下特征：
- `date`: 交易日期（YYYY-MM-DD）
- `open`, `high`, `low`, `close`: 开盘价、最高价、最低价、收盘价（已前复权）
- `volume`: 成交量（股或手）
- `amount`: 成交额（元）
- `outstanding_share` / `turnover`: 流通股本与换手率（新浪源特有字段，可用于构建流动性特征）

## 4. 离线数据预处理步骤（已落地执行）

为保证 `PortfolioEnv` 可以稳定读取并训练，我们在数据拉取完成后，执行了统一的离线预处理脚本：

- **脚本文件**：`data_thesis/preprocess_top20_offline_dataset.py`
- **输入目录**：`data_thesis/top20_weight_3years_data/`（20 只股票原始 CSV）
- **输出文件**：
  - `data_thesis/hs300_top20_offline_full.csv`
  - `data_thesis/hs300_top20_offline_train.csv`
  - `data_thesis/hs300_top20_offline_val.csv`
  - `data_thesis/hs300_top20_offline_test.csv`

### 4.1 预处理步骤

1. **逐文件读取与字段校验**  
   对 20 个股票文件逐一读取，检查是否包含 `date/open/high/low/close/volume/amount` 核心字段。

2. **时间排序与去重**  
   每只股票按 `date` 升序排序，并对重复日期去重（保留最后一条）。

3. **共同交易日对齐**  
   统计每个交易日的股票覆盖数，仅保留“20 只股票都存在数据”的日期，避免面板出现空洞。

4. **收益率计算**  
   对每只股票按时间计算 `return = close.pct_change()`，用于组合环境中的收益输入。

5. **首行空值处理**  
   由于收益率差分会导致每只股票首行 `return` 为空，统一删除对应行，保证训练输入无 NaN。

6. **时间序列切分**  
   按日期顺序切分 `train/val/test = 70%/15%/15%`，不打乱顺序，避免未来信息泄露。

### 4.2 这样做的原因（方法论解释）

- **为什么要检查每只股票是否有约 700+ 行**  
  近 3 年 A 股日频交易日大约在 720 个左右（扣除节假日和周末），若某只股票明显低于该量级，通常说明下载中断、文件损坏或数据缺失。700+ 只是快速质检阈值，不是硬编码规则。

- **为什么必须“共同交易日对齐”**  
  组合环境在同一时间步需要同时拿到 N 只股票的收益向量；若某些资产当日缺失，动作-奖励对应关系会被破坏，训练将不稳定甚至报错。

- **为什么使用收益率而不是价格直接训练**  
  收益率天然无量纲，便于跨资产比较和组合加权，也与组合收益公式 `w^T r` 直接对应，是组合优化和 RL 奖励设计的标准输入。

- **为什么按时间切分而不是随机切分**  
  金融时序数据强相关，随机切分会让未来信息泄露到训练集，导致评估结果虚高，不符合真实交易场景。

- **为什么拆分 train / val / test**  
  `train` 用于参数学习，`val` 用于调参和早停，`test` 用于最终无偏评估，满足毕业设计对实验严谨性的要求。

### 4.3 当前预处理结果（本次运行）

- 股票数量：20
- 对齐后交易日：723（约 3 年）
- 完整样本行数：14460
- 切分行数：
  - train：10120
  - val：2160
  - test：2180

以上结果已可直接接入 `PortfolioEnv` 与训练流程。

## 5. 后续任务规划建议

在此数据基础上，后续强化学习建模还可以进一步补充：
1. **技术指标衍生**：利用高开低收与成交量，计算 MA、MACD、RSI 等特征作为扩展状态输入。
2. **特征标准化**：对新增特征按训练集统计量做标准化，提升优化稳定性。
3. **评估基准统一**：固定与沪深300指数、等权组合、买入并持有等基准做对比，形成论文实验闭环。
