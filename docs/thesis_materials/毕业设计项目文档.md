## 项目概述

### 1.1 项目简介

本项目基于**Efficient Diffusion Policy (EDP)**，研究离线强化学习到在线强化学习的迁移方法，并将其应用于交易策略的构建。项目核心创新在于：

1. **利用扩散策略的强表示能力**进行离线预训练
2. **设计渐进式微调策略**实现平滑的离线到在线迁移
3. **解决分布偏移和灾难性遗忘**问题
4. **在交易场景中验证**方法的有效性

### 1.2 项目特色

- ✅ **理论基础扎实**：基于EDP和领域适应理论
- ✅ **方法创新明确**：渐进式微调 + 自适应学习率 + 知识蒸馏
- ✅ **实验设计完整**：多维度对比和消融实验
- ✅ **应用价值高**：面向实际交易场景

### 1.3 技术路线

```
离线预训练（EDP）
    ↓
保存策略参数
    ↓
在线微调（渐进式 + 自适应）
    ↓
评估与对比实验
```

## 研究背景与意义

### 2.1 研究背景

#### 2.1.1 强化学习在交易策略中的应用

**现状**：
- 强化学习在量化交易中应用广泛
- 传统方法（如DQN、PPO）在复杂市场环境中表现有限
- 需要更强的策略表示能力

**挑战**：
- 市场环境的非平稳性
- 历史数据和当前市场的分布差异
- 在线学习的样本效率问题

#### 2.1.2 离线强化学习的发展

**离线RL的优势**：
- 可以利用大量历史数据
- 不需要实时环境交互
- 训练稳定、可重复

**离线RL的局限**：
- 分布偏移问题：历史数据分布 ≠ 当前市场分布
- 策略可能无法适应市场变化
- 需要在线微调来适应新环境

#### 2.1.3 扩散模型在RL中的应用

**EDP的贡献**：
- 用扩散模型表示策略，表达能力更强
- 支持多模态动作分布
- 在D4RL benchmark上达到SOTA

**EDP的局限**：
- 主要针对离线RL场景
- 未考虑离线到在线的迁移问题

#### 4.2.2 渐进式微调策略（核心创新1）

**目标**：平滑地从离线数据过渡到在线数据

**方法设计**：

```python
class ProgressiveFinetuner:
    """
    渐进式微调器：逐步增加在线数据的比例
    """
    def __init__(self, offline_dataset, online_buffer):
        self.offline_dataset = offline_dataset
        self.online_buffer = online_buffer
        self.mix_ratio_schedule = {
            0: 1.0,      # 前1000步：100%离线数据
            1000: 0.8,   # 1000-5000步：80%离线 + 20%在线
            5000: 0.7,   # 5000-10000步：70%离线 + 30%在线
            10000: 0.5   # 10000步后：50%离线 + 50%在线
        }
    
    def sample_batch(self, step, batch_size):
        """根据当前步数采样混合batch"""
        mix_ratio = self._get_current_ratio(step)
        offline_size = int(batch_size * mix_ratio)
        online_size = batch_size - offline_size
        
        batch_offline = self.offline_dataset.sample(offline_size)
        batch_online = self.online_buffer.sample(online_size)
        
        return self._merge_batches(batch_offline, batch_online)
```

**理论依据**：
- 初始阶段：主要依赖离线知识，避免分布偏移
- 逐步过渡：平滑适应在线分布
- 最终阶段：平衡离线知识和在线经验

**优势**：
- 避免突然的分布变化
- 保持训练稳定性
- 充分利用离线数据

#### 4.2.3 自适应学习率调度（核心创新2）

**目标**：根据性能变化动态调整学习率

**方法设计**：

```python
class AdaptiveLRScheduler:
    """
    自适应学习率调度器：根据性能变化调整学习率
    """
    def __init__(self, initial_lr=3e-4, min_lr=1e-5, max_lr=1e-3):
        self.initial_lr = initial_lr
        self.current_lr = initial_lr
        self.min_lr = min_lr
        self.max_lr = max_lr
        self.performance_history = []
        self.patience = 10  # 性能不提升的容忍步数
    
    def update(self, current_performance):
        """根据性能变化调整学习率"""
        self.performance_history.append(current_performance)
        
        if len(self.performance_history) < 20:
            return self.current_lr
        
        # 计算最近10步的性能趋势
        recent_perf = np.mean(self.performance_history[-10:])
        previous_perf = np.mean(self.performance_history[-20:-10])
        trend = recent_perf - previous_perf
        
        if trend > 0.01:  # 性能提升明显
            # 保持或略微增加学习率
            self.current_lr = min(self.current_lr * 1.05, self.max_lr)
        elif trend < -0.01:  # 性能下降
            # 降低学习率
            self.current_lr = max(self.current_lr * 0.95, self.min_lr)
        # else: 保持当前学习率
        
        return self.current_lr
```

**理论依据**：
- 性能提升时：可以保持或增加学习率，加快学习
- 性能下降时：降低学习率，提高稳定性
- 自适应调整：不需要手动调参

**优势**：
- 自动适应训练过程
- 提高训练效率
- 减少超参数调优

#### 4.2.4 知识蒸馏机制（核心创新3）

**目标**：防止在线微调时忘记离线知识

**方法设计**：

```python
class KnowledgeDistilledFinetuner:
    """
    知识蒸馏微调器：用离线策略指导在线学习
    """
    def __init__(self, offline_policy, online_policy, distill_coef=0.1):
        self.offline_policy = offline_policy  # 冻结的离线策略
        self.online_policy = online_policy    # 正在微调的在线策略
        self.distill_coef = distill_coef
    
    def compute_distillation_loss(self, observations):
        """计算知识蒸馏损失"""
        # 离线策略的动作分布（作为"教师"）
        offline_actions = self.offline_policy.sample(observations, num_samples=10)
        offline_mean = np.mean(offline_actions, axis=0)
        offline_std = np.std(offline_actions, axis=0)
        
        # 在线策略的动作分布（作为"学生"）
        online_mean = self.online_policy.get_mean(observations)
        online_std = self.online_policy.get_std(observations)
        
        # KL散度：让在线策略学习离线策略的分布
        kl_loss = self._compute_kl_divergence(
            (online_mean, online_std),
            (offline_mean, offline_std)
        )
        
        return kl_loss
    
    def train_step(self, batch):
        """训练步骤：RL loss + 蒸馏loss"""
        # 标准的RL loss（SAC/TD3）
        rl_loss = self._compute_rl_loss(batch)
        
        # 知识蒸馏loss
        distillation_loss = self.compute_distillation_loss(
            batch['observations']
        )
        
        # 总loss
        total_loss = rl_loss + self.distill_coef * distillation_loss
        
        return total_loss, {
            'rl_loss': rl_loss,
            'distillation_loss': distillation_loss,
            'total_loss': total_loss
        }
```

**理论依据**：
- 知识蒸馏：用"教师"网络（离线策略）指导"学生"网络（在线策略）
- KL散度：衡量两个策略分布的差异
- 正则化：防止在线策略偏离离线策略太远

**优势**：
- 防止灾难性遗忘
- 保持离线知识
- 提高训练稳定性

#### 4.2.5 分布偏移度量与自适应策略

**目标**：根据分布偏移程度调整微调策略

**方法设计**：

```python
class DistributionShiftAnalyzer:
    """
    分布偏移分析器：度量离线数据和在线数据的分布差异
    """
    def __init__(self):
        self.offline_stats = None
        self.online_stats = None
    
    def compute_shift(self, offline_data, online_data):
        """计算分布偏移程度"""
        # 方法1：KL散度
        kl_div = self._compute_kl_divergence(
            offline_data['actions'],
            online_data['actions']
        )
        
        # 方法2：MMD距离
        mmd_dist = self._compute_mmd(
            offline_data['observations'],
            online_data['observations']
        )
        
        # 方法3：动作分布差异
        action_diff = self._compute_action_distribution_diff(
            offline_data['actions'],
            online_data['actions']
        )
        
        # 综合偏移程度（归一化到[0, 1]）
        shift_score = (kl_div + mmd_dist + action_diff) / 3
        
        return shift_score
    
    def get_finetuning_strategy(self, shift_score):
        """根据偏移程度返回微调策略"""
        if shift_score < 0.3:
            # 低偏移：保守微调
            return {
                'lr': 1e-4,
                'mix_ratio': 0.8,  # 更多离线数据
                'distill_coef': 0.2  # 更强的蒸馏
            }
        elif shift_score < 0.6:
            # 中等偏移：正常微调
            return {
                'lr': 3e-4,
                'mix_ratio': 0.7,
                'distill_coef': 0.1
            }
        else:
            # 高偏移：激进微调
            return {
                'lr': 5e-4,
                'mix_ratio': 0.5,  # 更多在线数据
                'distill_coef': 0.05  # 更弱的蒸馏
            }
```

**理论依据**：
- 分布偏移度量：量化离线数据和在线数据的差异
- 自适应策略：根据偏移程度调整微调参数
- 平衡稳定性和适应性

**优势**：
- 自动适应不同场景
- 提高方法的鲁棒性
- 减少手动调参

#### 算法1：离线预训练

```

    4.   loss = agent.train(batch)
    5.   更新参数 θ
6. 保存 θ_offline
```

#### 算法2：在线微调（完整流程）

```
输入：预训练参数 θ_offline，在线环境 env
输出：微调后的策略参数 θ_final

1. 加载 θ_offline，初始化在线策略 θ_online = θ_offline
2. 初始化在线replay buffer B_online
3. 初始化渐进式微调器 P，自适应学习率调度器 A，知识蒸馏器 K
4. 
5. for step in range(n_online_steps):
    6.   # 收集在线数据
    7.   action = π_online(s)
    8.   s', r, done = env.step(action)
    9.   B_online.add(s, a, r, s', done)
    10.  
    11.  # 每N步更新一次
    12.  if step % update_freq == 0:
    13.      # 计算分布偏移
    14.      shift_score = compute_distribution_shift(D_offline, B_online)
    15.      strategy = get_finetuning_strategy(shift_score)
    16.      
    17.      # 渐进式采样
    18.      batch = P.sample_batch(step, batch_size)
    19.      
    20.      # 自适应学习率
    21.      lr = A.update(current_performance)
    22.      
    23.      # 训练（RL loss + 蒸馏loss）
    24.      rl_loss = compute_rl_loss(batch)
    25.      distill_loss = K.compute_distillation_loss(batch['obs'])
    26.      total_loss = rl_loss + strategy['distill_coef'] * distill_loss
    27.      
    28.      # 更新参数
    29.      θ_online = update(θ_online, total_loss, lr)
30. 
31. return θ_online
```

### 5.1 实验环境设置

#### 5.1.1 交易环境设计

**环境特点**：
- 模拟真实交易场景
- 支持多种市场条件（牛市、熊市、震荡市）
- 可配置的分布偏移程度

**状态空间（主实验：多标的）**：
- **截面特征**：股票池中各标的的收益率、成交量、技术指标等（按日对齐为矩阵或展平向量，论文中固定一种表示）。
- **组合状态**：上一期组合权重 $\mathbf{w}_{t-1}$；可选：组合净值、现金比例等。
- **历史信息**：可选过去 $N$ 步市场状态（维数随 $N$、标的数上升，需控制规模）。

**动作空间（主实验：多标的）**：
- **连续动作（默认）**：组合权重向量 $\mathbf{w}_t$（维数 = 股票池大小 $N$）；须在环境中施加**可行域约束**（如 long-only + 单纯形归一、或带杠杆的盒约束），并与行为策略、离线数据构造一致。
- **离散动作**：一般不作为主实验默认（维数爆炸）；若采用需说明离散化方式。

**奖励函数（主实验：组合；单标的为 $N=1$ 特例）**：
- 训练阶段采用**逐步标量奖励**（非直接用 Sharpe 作为每步 reward）。

**多标的（主实验）**：令 $\mathbf{w}_t$ 为组合权重，$\mathbf{r}_t$ 为各标的当日简单收益向量，则：

$$
r_t^{(port)}=\mathbf{w}_{t-1}^{\top}\mathbf{r}_t-\lambda\|\mathbf{w}_t-\mathbf{w}_{t-1}\|_1
$$

- 第一项为组合收益，第二项为换仓成本近似；$\lambda$ 与费率设定对齐。
- 须保证 **MDP 因果**：决策基于 $t$ 日开盘前可得信息时，应对 $\mathbf{r}_t$、特征与 $\mathbf{w}_{t-1}$ 的时序作严格定义，避免前视。

**单标的（可选对照）**：

$$
r_t = w_{t-1}\cdot R_t - c\cdot |w_t - w_{t-1}|
$$

$w_t\in[-1,1]$，$R_t$ 为标的简单收益。

**说明**：

- Sharpe、最大回撤等用于**验证/测试与选优**（见 5.3）。
- 若更换奖励或约束，须**重算离线 reward**、**保持时间切分不变**，并在论文中说明。
- **扩散策略与网络**：动作维 = $N$（或经参数化后的维数），需在实现中写明维度与 **EDP** 训练配置。

- **主实验**：国内、**日频**、**固定股票池（约 20～50 只，论文中写明筛选规则）**；构建 **面板数据**（按日期对齐、缺失处理），按时间划分 **离线训练段 / 验证段 / 测试段**（或 walk-forward），避免未来信息泄漏。

### 5.2 对比实验设计

#### 5.2.0 轻量基线的理论定义（当前已落地）

为在正式 EDP 训练前验证数据管线与评估闭环，本文采用一个低复杂度可复现的轻量基线。该基线不是深度强化学习，而是**无模型的黑盒策略搜索（Evolution Strategy, ES）**。

**策略参数化**

- 设上一期各资产收益向量为 \(\mathbf{r}_{t-1}\)，策略参数为 \(\boldsymbol{\theta}\in\mathbb{R}^{N}\)。
- 当前权重由 softmax 给出：

$$
\mathbf{w}_t=\operatorname{softmax}\!\left(\boldsymbol{\theta}\odot \mathbf{r}_{t-1}\right)
$$

其中 \(\odot\) 为逐元素乘法。该形式可解释为“按过去收益做动量型加权”。

**奖励函数（与组合环境一致）**

$$
r_t=\mathbf{w}_{t-1}^{\top}\mathbf{r}_t-\lambda\left\|\mathbf{w}_t-\mathbf{w}_{t-1}\right\|_1
$$

- 第一项为组合当期收益；
- 第二项为换仓成本近似（\(\lambda\) 对应交易费率）。

**优化方法（ES 梯度估计）**

- 对参数加入高斯扰动：

$$
\boldsymbol{\theta}_i=\boldsymbol{\theta}+\sigma\boldsymbol{\epsilon}_i,\quad \boldsymbol{\epsilon}_i\sim\mathcal{N}(0,\mathbf{I})
$$

- 评估每个扰动参数在训练段的目标值 \(J(\boldsymbol{\theta}_i)\)；
- 将得分标准化后估计梯度并更新参数：

$$
\nabla_{\theta}J \approx \frac{1}{\sigma}\frac{1}{K}\sum_{i=1}^{K}\hat{s}_i\boldsymbol{\epsilon}_i,\qquad
\boldsymbol{\theta}\leftarrow \boldsymbol{\theta}+\alpha\nabla_{\theta}J
$$

其中 \(K\) 为种群大小（`pop_size`），\(\sigma\) 为扰动强度，\(\alpha\) 为更新步长（`lr`）。

**定位与作用**

- 该基线用于提供一个“可解释、低成本、可快速复现”的对照；
- 其结果主要用于验证流程正确性与形成最小对照，并不替代 EDP 作为论文主方法；
- 后续正式结论仍以 EDP 与市场/方法基线的对比为主体。

### 5.3 评估指标

本节指标主要用于**验证/测试阶段评估、论文图表与模型选优**；训练仍以第 5.1 节约定的逐步奖励 \(r_t\) 与 RL/扩散损失为主（见 1.4 节）。

#### 5.3.1 性能指标

1. **归一化收益（Normalized Return）**
   - 定义：相对于专家策略的归一化分数
   - 计算：使用类似D4RL的归一化方法
   - 范围：[0, 1]，1表示专家水平

2. **Sharpe Ratio**
   - 定义：收益风险比
   - 计算：$\text{Sharpe} = \frac{\text{平均收益}}{\text{收益标准差}}$
   - 用途：衡量风险调整后的收益

3. **最大回撤（Max Drawdown）**
   - 定义：从峰值到谷值的最大跌幅
   - 计算：$\text{MD} = \max_{t} \frac{\text{Peak}_t - \text{Value}_t}{\text{Peak}_t}$
   - 用途：衡量策略的风险

4. **累计收益（Cumulative Return）**
   - 定义：整个测试期间的总收益
   - 计算：$\sum_{t=0}^{T} r_t$
   - 用途：衡量策略的整体盈利能力

---

#### 5.3.2 效率指标

1. **样本效率（Sample Efficiency）**
   - 定义：达到目标性能需要的样本数
   - 计算：达到目标性能的步数
   - 用途：衡量数据利用效率

2. **收敛速度（Convergence Speed）**
   - 定义：性能提升的速度
   - 计算：性能曲线的斜率或达到收敛的步数
   - 用途：衡量训练效率

3. **训练时间（Training Time）**
   - 定义：实际训练时间（秒/小时）
   - 用途：衡量计算成本

---

#### 5.3.3 稳定性指标

1. **性能方差（Performance Variance）**
   - 定义：多次运行的性能标准差
   - 计算：$\sigma(\text{returns})$
   - 用途：衡量方法的稳定性

2. **成功率（Success Rate）**
   - 定义：达到目标性能的运行比例
   - 计算：$\frac{\text{成功运行数}}{\text{总运行数}}$
   - 用途：衡量方法的可靠性

3. **不同环境下的表现**
   - 定义：在不同市场环境下的性能
   - 用途：衡量方法的泛化能力

---

- 已完成 `PortfolioEnv` 读取适配：支持从 split 长表（`date/symbol/return`）直接构建收益率面板。

#### 5.5.2 下一步 Step 1：最小训练跑通（Sanity Check）

- **输入数据**：`hs300_top20_offline_train.csv`
- **执行目标**：先跑小规模训练（少量 epoch/step），只验证链路正确性，不追求收益最优。
- **通过标准**：
  - 维度匹配（资产数=20，动作维一致）
  - 训练过程无 NaN/崩溃
  - Loss 和奖励曲线可解释（至少不异常发散）

> 作用：先排除工程问题，避免长时间训练后才发现管线错误。

#### 5.5.3 下一步 Step 2：验证集调参与模型选择

- **输入数据**：`hs300_top20_offline_val.csv`
- **重点超参数**：
  - 学习率（learning rate）
  - 交易成本参数（fee_rate）
  - softmax 温度（softmax_temp）
  - batch size、更新步数
- **目标**：选出泛化更好的参数组合，而不是训练集最好分数。

> 作用：将调参行为限制在验证集，保护测试集客观性。

#### 5.5.4 下一步 Step 3：测试集一次性评估

- **输入数据**：`hs300_top20_offline_test.csv`
- **原则**：冻结超参数后仅评估，不在 test 上反复试错。
- **输出指标**：
  - 累计收益率
  - 年化收益率
  - 年化波动率
  - 夏普比率
  - 最大回撤（MDD）
  - 换手率（Turnover）

> 作用：模拟真实未知市场的最终评测，作为论文主结果。

#### 5.5.5 下一步 Step 4：基准策略对比

建议至少设置以下 baseline：

1. 沪深300（或 510300）Buy & Hold  
2. Top20 等权静态组合（Equal Weight）  
3. （可选）简单动量/均值回复规则策略

> 作用：证明方法有效性来自“相对改进”，而非单条收益曲线的偶然波动。

#### 5.5.6 下一步 Step 5：论文图表与结果沉淀

建议固定输出以下结果用于正文与答辩：

- 净值曲线（策略 vs baseline）
- 回撤曲线（Drawdown）
- 指标汇总表（收益/风险/换手）
- 关键超参数表与实验设置表（可复现）

> 作用：形成“可复现 + 可解释 + 可比较”的完整实验闭环。

### B. 关键技术难点

#### 难点1：在线数据收集

**挑战**：
- 交易环境的实时性要求
- 数据质量保证
- 与环境的交互接口

**解决方案**：
- 使用模拟环境先验证
- 设计标准化的环境接口
- 实现数据验证机制

---

#### 难点2：分布偏移度量

**挑战**：
- 如何准确度量分布差异
- 如何设计自适应策略
- 计算效率问题

**解决方案**：
- 使用KL散度、MMD等成熟方法
- 设计简化的度量指标
- 使用采样方法降低计算量

---

#### 难点3：灾难性遗忘

**挑战**：
- 在线微调可能忘记离线知识
- 如何平衡新旧知识
- 知识蒸馏的效果

**解决方案**：
- 使用知识蒸馏机制
- 混合使用离线数据
- 设计正则化项
