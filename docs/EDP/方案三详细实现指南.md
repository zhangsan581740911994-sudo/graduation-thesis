## 理论基础

### 1.1 领域适应（Domain Adaptation）基础

**定义**：
领域适应是迁移学习的一个分支，旨在解决**源域（source domain）**和**目标域（target domain）**之间的分布差异问题。

**在离线到在线RL中的含义**：
- **源域（离线）**：历史数据分布 $P_{\text{offline}}(s, a)$
- **目标域（在线）**：当前环境分布 $P_{\text{online}}(s, a)$
- **问题**：$P_{\text{offline}} \neq P_{\text{online}}$，导致策略性能下降

**核心挑战**：
1. **分布偏移（Distribution Shift）**：离线数据和在线数据的分布不同
2. **Bootstrap Error**：Q值在OOD（Out-of-Distribution）动作上不准确
3. **灾难性遗忘**：在线微调可能忘记离线学到的知识

---

### 1.2 Actor-Critic Alignment (ACA)

**论文**：Actor-Critic Alignment for Offline-to-Online Reinforcement Learning (ICML 2023)

**核心思想**：
不直接对齐特征分布，而是通过**对齐Q值与离线策略**来解决分布偏移。

**关键洞察**：
- 传统方法：对齐特征分布 → 可能扭曲特征空间
- ACA方法：对齐Q值 → 约束OOD动作的Q值，保持特征空间结构

**数学表达**：
$$
Q_{\text{aligned}}(s, a) = \begin{cases}
Q(s, a) & \text{if } \pi_{\text{offline}}(a|s) > \tau \\
Q(s, a) \cdot \lambda & \text{if } \pi_{\text{offline}}(a|s) \leq \tau
\end{cases}
$$

其中：
- $\pi_{\text{offline}}(a|s)$：离线策略的动作概率
- $\tau$：阈值
- $\lambda < 1$：惩罚系数

---

### 1.3 EDP中的动作分布（action_dist）

**EDP的关键组件**：
在 `diffusion/dql.py` 的 `get_diff_terms()` 函数中，EDP构造了一个**高斯动作分布**：

```python
# 伪代码（实际在get_diff_terms中）
pred_astart = self.diffusion.p_mean_variance(...)["pred_xstart"]
action_dist = self.policy_dist.apply(params['policy_dist'], pred_astart)
log_prob = action_dist.log_prob(sample)
```

**为什么重要**：
- `action_dist` 提供了**可计算的 log π(a|s)**
- 这正是领域适应需要的：判断动作是否在离线策略分布内
- 不需要额外的判别器，直接利用EDP的现有组件

---

### 1.4 分布匹配理论

**KL散度**：
$$
D_{KL}(P||Q) = \mathbb{E}_{x \sim P}[\log \frac{P(x)}{Q(x)}]
$$

**在方案三中的应用**：
- 目标：最小化在线策略和离线策略的KL散度
- 方法：通过Q值对齐间接实现分布匹配
- 优势：不需要显式计算KL散度，通过权重实现

## 核心思想

### 2.1 问题定义

**输入**：
- 离线数据集 $\mathcal{D}_{\text{offline}} = \{(s_i, a_i, r_i, s'_i)\}$
- 离线训练好的策略 $\pi_{\text{offline}}$（EDP）
- 在线环境（可以交互）

**目标**：
- 在保持离线策略知识的前提下，适应在线环境
- 避免分布偏移导致的性能下降

---

### 2.2 解决方案概述

**核心机制**：Q值对齐

1. **保存离线策略快照**：
   - 离线训练完成后，保存策略参数和action_dist参数

2. **对齐Q值**：
   - 对于每个动作，计算其在离线策略下的log_prob
   - 如果log_prob低（OOD动作），降低其Q值
   - 如果log_prob高（ID动作），保持Q值

3. **渐进式适应**：
   - 初始阶段：强对齐（alignment_coef = 1.0）
   - 逐步降低对齐强度（alignment_coef *= 0.95）
   - 最终：弱对齐或不对齐

---

### 2.3 与EDP的关系

**完全基于EDP**：
- ✅ 使用EDP的DiffusionPolicy（扩散策略）
- ✅ 使用EDP的action_dist（动作分布）
- ✅ 使用EDP的get_diff_loss（扩散损失）
- ✅ 使用EDP的guide_loss（TD3/CRR/IQL引导损失）

**新增内容**：
- ⭐ 对齐机制：`align_q_with_offline_policy()`
- ⭐ 快照保存：`save_offline_policy_snapshot()`
- ⭐ 混合训练：离线数据 + 在线数据

**完整代码**：

```python
def align_q_with_offline_policy(self, q_value, action, observation):
    """
    核心对齐函数：如果动作在离线策略分布外，降低其Q值。
    
    参数：
        q_value: 原始Q值，shape=(batch_size,)
        action: 当前动作，shape=(batch_size, action_dim)
        observation: 当前观测，shape=(batch_size, obs_dim)
    
    返回：
        aligned_q: 对齐后的Q值，shape=(batch_size,)
    """
    # 如果未启用对齐或没有快照，直接返回原始Q值
    if not self.config.use_alignment or self.offline_policy_snapshot is None:
        return q_value
    
    # 步骤1：用离线策略计算动作的log_prob
    # 1.1 用离线策略预测动作（均值）
    offline_pred_action = self.policy.apply(
        self.offline_policy_snapshot['policy_params'],
        observation,
        method=self.policy.__call__  # 调用采样方法，但这里我们需要均值
    )
    
    # 注意：上面的方法可能不对，因为DiffusionPolicy的__call__是采样
    # 更好的方法是直接用pred_astart的逻辑
    # 这里简化处理：假设我们已经有了offline_pred_action
    
    # 1.2 构造离线动作分布
    offline_action_dist = self.policy_dist.apply(
        self.offline_policy_snapshot['policy_dist_params'],
        offline_pred_action
    )
    
    # 1.3 计算动作在离线策略下的log_prob
    offline_log_prob = offline_action_dist.log_prob(action)
    
    # 步骤2：计算对齐权重
    # log_prob越高（动作越在分布内），权重越大
    temperature = self.config.alignment_temperature
    alignment_weight = jnp.exp(offline_log_prob / temperature)
    # 裁剪到[0, 1]
    alignment_weight = jnp.clip(alignment_weight, 0, 1)
    
    # 步骤3：对齐Q值
    # OOD动作的Q值被降低，ID动作的Q值保持
    penalty_factor = self.config.alignment_penalty
    aligned_q = (
        q_value * alignment_weight + 
        q_value * (1 - alignment_weight) * penalty_factor
    )
    
    return aligned_q
```

**关键点**：
- 使用EDP的`policy_dist`计算log_prob
- 通过`alignment_weight`控制对齐强度
- `penalty_factor`控制OOD动作的惩罚程度

```python
    """
    构造一个「value_loss_fn」，用于计算当前 batch 下 Q（和 V）的 TD 损失。
    
    修改：添加对齐机制支持。
    """
    def value_loss_fn(params, tgt_params, rng):
        observations = batch['observations']
        actions = batch['actions']
        rewards = batch['rewards']
        next_observations = batch['next_observations']
        dones = batch['dones']
        
        # 计算target Q值（标准流程）
        if self.config.max_q_backup:
            # ... 原有的max_q_backup逻辑 ...
            # 注意：这里也需要对齐
            pass
        else:
            next_action = self.policy.apply(
                tgt_params['policy'], rng, next_observations
            )
            next_action = jnp.clip(next_action, -self.max_action, self.max_action)
            tgt_q1 = self.qf.apply(
                tgt_params['qf1'], next_observations, next_action
            )
            tgt_q2 = self.qf.apply(
                tgt_params['qf2'], next_observations, next_action
            )
            tgt_q = jnp.minimum(tgt_q1, tgt_q2)
        
        tgt_q = rewards + (1 - dones) * self.config.discount * tgt_q
        
        # ⭐ 关键修改：对齐target Q值
        if self.config.use_alignment and self.offline_policy_snapshot is not None:
            # 计算对齐系数（可渐进调整）
            alignment_coef = self.config.alignment_coef
            
            # 对齐target Q值
            aligned_tgt_q = self.align_q_with_offline_policy(
                tgt_q, next_action, next_observations
            )
            
            # 混合原始Q值和对齐后的Q值
            tgt_q = (1 - alignment_coef) * tgt_q + alignment_coef * aligned_tgt_q
        
        tgt_q = jax.lax.stop_gradient(tgt_q)
        
        # 计算当前Q值
        cur_q1 = self.qf.apply(params['qf1'], observations, actions)
        cur_q2 = self.qf.apply(params['qf2'], observations, actions)
        
        # qf loss
        qf1_loss = mse_loss(cur_q1, tgt_q)
        qf2_loss = mse_loss(cur_q2, tgt_q)
        
        qf_loss = qf1_loss + qf2_loss
        return (qf1_loss, qf2_loss), locals()
    
    return value_loss_fn
```

**关键修改点**：
- 在计算target Q值后，添加对齐逻辑
- 使用`alignment_coef`控制对齐强度（可渐进调整）
- 对齐只应用于target Q值，不影响当前Q值的计算
