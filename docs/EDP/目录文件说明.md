## 目录与文件说明（按执行顺序和理论顺序）

> 作用：帮你快速知道「每个文件是干什么的、在整个 EDP 流程里处于哪一环、里面有哪些你需要重点理解的函数/类」。

---

### 一、整体执行顺序（从命令行到评估）

1. **入口与整体流程控制**  
   - `diffusion/trainer.py`  
     - 从命令行 `python -m diffusion.trainer ...` 进来后，建 `DiffusionTrainer`，调用 `train()`。  
     - 在 `train()` 里一次性建好：logger、dataset、env 封装（eval_sampler）、policy（扩散策略）、Q/V 网络、DiffusionQL agent、SamplerPolicy。  
     - 然后进入训练循环：每步 `dataset.sample()` 抽 batch → `agent.train(batch)` 更新参数；按周期用 `SamplerPolicy` 在 env 里评估（生成动作 + 跑轨迹 + 记 normalized return）。

2. **算法核心（扩散 + TD3/CRR/IQL）**  
   - `diffusion/dql.py`  
     - 定义 **DiffusionQL**：把扩散策略 `policy`、Q 网络 `qf1/qf2`、V 网络 `vf`（IQL 用）、高斯策略 `policy_dist` 组装在一起。  
     - 提供一步训练接口：根据 `loss_type`（TD3/CRR/IQL）调用对应 `_train_step_*`，在扩散 MSE 的基础上加上各自的 value loss + guide loss。

3. **扩散模型与采样**  
   - `diffusion/diffusion.py`  
     - 定义 **GaussianDiffusion**：β/α 调度、前向加噪 $q(x_t\|x_0)$、反向近似 $p_\theta(x_{t-1}\|x_t)$、`training_losses`（ELBO→噪声 MSE）、DDPM/DDIM 等采样函数。  
   - `diffusion/dpm_solver.py`  
     - 实现 **DPM-Solver**：少步 ODE 采样，用更少步数替代 DDPM 的满 T 步。

4. **网络结构（策略 / Q / V / 高斯头）**  
   - `diffusion/nets.py`  
     - 定义 **DiffusionPolicy**（扩散策略网络，里面用 `GaussianDiffusion` 做加噪/去噪）、`PolicyNet`（MLP+时间嵌入）、`Critic`（Q 网络）、`Value`（V 网络）、`GaussianPolicy`（高斯头，用于构造 log π）。

5. **数据与采样器**  
   - `data/dataset.py`、`utilities/replay_buffer.py`、`utilities/traj_dataset.py`、`utilities/sampler.py`  
     - 负责从 D4RL / RLUP 读离线数据、做 n-step、封装成 `Dataset`，以及在 env 中滚轨迹（`TrajSampler`）。

6. **配置与常量**  
   - `diffusion/hps.py`、`diffusion/constants.py`、`WORKSPACE` 等  
     - 存 env 对应的默认超参（lr、max_grad_norm 等）和 env/dataset 名字映射，Bazel/依赖配置等。

### 二、按流程细分：每个文件的职责与你需要看的关键部分

#### 1. `diffusion/trainer.py` —— 入口 + 训练/评估主流程

- **职责**：  
  - 解析命令行 / FLAGS；  
  - 一次性建好 logger、数据集、env 封装（eval_sampler）、policy/Q/V/agent/SamplerPolicy；  
  - 跑训练循环和评估循环，记录 normalized return 等指标。

- **你需要理解的关键点**：
  - `DiffusionTrainer.__init__`：  
    - 只做「读 FLAGS、选激活函数、写回一些超参（max_grad_norm、lr_decay_steps）」；  
    - 不建数据/网络（真正建在 `_setup()` 里）。
  - `DiffusionTrainer.train`：  
    - 第一行 `self._setup()`：调用 `_setup_logger/_setup_dataset/_setup_policy/_setup_qf/_setup_vf`，建好所有组件；  
    - 然后是 epoch 循环：  
      - 训练阶段：`batch = self._dataset.sample()` → `metrics.update(self._agent.train(batch))`；  
      - 评估阶段：对每个 `act_method` 设置 `self._sampler_policy.act_method`，用 `self._eval_sampler.sample(...)` 滚轨迹，算 normalized return。
  - `_setup_dataset` + `_setup_d4rl/_setup_rlup`：  
    - 用 D4RL / RLUP 把离线数据读到内存，封装成 `Dataset`；  
    - 同时建一个 `TrajSampler(env, max_traj_length)` 作为评估环境。  
  - `_setup_policy/_setup_qf/_setup_vf/_algo`：  
    - 分别建 `DiffusionPolicy`、`Critic/Value`、`DiffusionQL`。  
  - `class SamplerPolicy`：  
    - 评估时「obs → action」的封装：  
      - `ddpm_act/dpm_act/ddim_act`：单采样动作；  
      - `ensemble_act/ddpmensemble_act/dpmensemble_act`：多采样 + 用 Q 选（EAS）。

#### 2. `diffusion/dql.py` —— DiffusionQL：扩散 + TD3/CRR/IQL 的组合

- **职责**：  
  - 把扩散策略 `policy`、Q 网络 `qf1/qf2`、V 网络 `vf`、高斯策略 `policy_dist` 打包成一个离线 RL 算法；  
  - 在扩散 MSE 的基础上，加上 TD3 / CRR / IQL 的 value loss 和 guide loss，完成一次参数更新。

- **你需要理解的关键点**（已在代码中加了注释）：
  - 顶部：`class DiffusionQL(Algo)` 的整体 docstring（说明它装了哪些模块、提供哪些接口）。  
  - `get_default_config`：  
    - 超参大致分三类：  
      - 离线 RL 基本配置：`nstep`、`discount`、`tau`、`policy_tgt_freq`。  
      - 扩散相关：`num_timesteps`、`schedule_name`、`time_embed_size`。  
      - RL 头相关：`loss_type`（TD3/CRR/IQL）、`alpha`、`use_pred_astart`（action approximation）、CRR/IQL 的 expectile / awr_temperature 等。  
  - `__init__`：  
    - 用 `policy/qf/vf/policy_dist` 的 `init` 建立 TrainState：`_train_states['policy/qf1/qf2/vf/policy_dist']`；  
    - 拷一份 target 参数 `_tgt_params`（policy, qf1, qf2, vf）；  
    - 记录 observation_dim / action_dim / max_action。  
  - `get_value_loss(batch)`：  
    - 构造一个 value_loss_fn：给 batch 的 (s,a,r,s') 算 target Q = r + γ·min(Q1',Q2')，当前 Q1/Q2(s,a)，loss = 两个 MSE 之和。  
  - `get_diff_terms(...)` / `get_diff_loss(batch)`：  
    - 把扩散训练那块打包：  
      - **对真实动作 a（=x₀）加噪** → x_t；  
      - 用策略网络预测噪声/预测 x₀，算每个样本的 diff loss（MSE）；  
      - 若 `use_pred_astart=True`：用 `p_mean_variance(..., pred_xstart)` 得到当前步预测的 x₀（pred_astart）——**这里就是「训练加速：action approximation」真正落地的地方**：训练算 guide loss 时直接用 pred_astart，当作“当前策略输出的动作”，而不是每次都跑完整反向链采样；  
      - 构造近似动作分布 `action_dist` / `log_p`，供 CRR/IQL 用。  
    - `get_diff_loss` 再封装出 diff_loss_fn，返回平均 diff_loss 和 pred_astart 等。
  - `_train_step`：  
    - 根据 `loss_type` 分发到 `_train_step_td3/_crr/_iql`，是真正“一步训练”的入口。
  - `_train_step_td3/_train_step_crr/_train_step_iql`：  
    - 三个分支的区别主要体现在 **guide loss** 上：  
      - TD3：guide = −λ·Q(s, pred_astart)；  
      - CRR：guide = −E[λ(A)·log π(a|s)]（优势加权行为克隆）；  
      - IQL：guide = AWR 权重 exp((Q−V)/τ)·log π(a|s)。  
    - 三者都用 **同一个扩散 diff_loss**，只是 guide loss 定义不同；并且都复用了上面 `get_diff_loss` 里算出的 **pred_astart**，共同构成「**训练加速 + 不同算法的兼容性**」这一层。

#### 3. `diffusion/diffusion.py` —— GaussianDiffusion：扩散前向/后向 + 训练 + 采样

- **职责**：  
  - 定义前向加噪 $q(x_t\|x_0)$ 和后验 $q(x_{t-1}\|x_t,x_0)$ 的闭式公式；  
  - 定义反向近似 $p_\theta(x_{t-1}\|x_t)$ 的形式（用网络输出的均值/噪声来参数化）；  
  - 提供训练 loss（ELBO→噪声 MSE）和采样函数（DDPM/DDIM）。

- **你需要理解的关键点**：
  - `GaussianDiffusion.__init__`：  
    - 如何根据 `num_timesteps`、`schedule_name` 构造 β/α/ᾱ；  
    - 这些表在训练和生成时都用到，但本身不参与训练（固定 schedule）。  
  - `q_sample`：  
    - 实现 $x_t = \sqrt{\bar\alpha_t} x_0 + \sqrt{1-\bar\alpha_t}\epsilon$，是「前向加噪」。  
  - `q_posterior_mean_variance`：  
    - 实现 $q(x_{t-1}\|x_t,x_0)$ 的均值/方差，是前向后验（训练时当作目标）。  
  - `p_mean_variance`：  
    - 用网络输出的 `model_output`（预测噪声/预测 x₀）+ q 的后验公式构造 $p_\theta(x_{t-1}\|x_t)$ 的均值/方差；  
    - 同时给出 `pred_xstart`，供 action approximation 使用。  
  - `training_losses`：  
    - 实现 ELBO 在固定方差下化简后的噪声 MSE：$\|\epsilon - \epsilon_\theta(x_t,t)\|^2$。  
    - 这是「扩散训练」的核心部分。

#### 4. `diffusion/nets.py` —— 策略/价值网络 + 采样接口

- **职责**：  
  - 搭「网络骨架」：  
    - `PolicyNet`：以 (state, action, time_embed) 为输入的 MLP。  
    - `DiffusionPolicy`：封装 `PolicyNet` + `GaussianDiffusion`，提供：  
      - `__call__` / `loss` 用于训练（预测噪声/损失）；  
      - `ddpm_sample/ddim_sample/dpm_sample` 用于生成动作。  
    - `Critic`：Q 网络；`Value`：V 网络；`GaussianPolicy`：高斯策略头。

- **你需要理解的关键点**：
  - `TimeEmbedding`：时间 t 的嵌入方式（sin/cos + MLP）。  
  - `PolicyNet.__call__`：state、action、time_embed 如何拼在一起过 MLP。  
  - `DiffusionPolicy.loss`：调用 `GaussianDiffusion.training_losses`，是训练时的入口。  
  - `DiffusionPolicy.ddpm_sample/dpm_sample`：调用 `GaussianDiffusion` 或 `DPM_Solver` 进行采样，是生成时的入口。

#### 5. `data/dataset.py` / `utilities/replay_buffer.py` / `utilities/traj_dataset.py` / `utilities/sampler.py`

- **职责**：  
  - `data/dataset.py`：  
    - `Dataset`：把 dict 形式的数据（obs/actions/rewards/next_obs/dones）封装成对象，支持 `set_sampler` 和 `sample()`；  
    - `RLUPDataset`：从 RL Unplugged 读数据。  
  - `utilities/replay_buffer.py`：  
    - `get_d4rl_dataset(env, nstep, gamma, norm_reward)`：从 D4RL env 读离线数据，做 n-step/归一化，返回 dict；  
    - `ReplayBuffer`：通用的 buffer 封装。  
  - `utilities/traj_dataset.py`：  
    - `get_nstep_dataset`：按轨迹做 n-step return、排序等。  
  - `utilities/sampler.py`：  
    - `TrajSampler`：给定 env 和 policy，滚多条轨迹，返回 `{observations, actions, rewards, dones,...}` 序列，评估时用。

#### 6. 其他配置/常量

- `diffusion/hps.py`：为不同 env 提供默认超参（learning rate、max_grad_norm、训练轮数等）。  
- `diffusion/constants.py`：ENV_MAP、DATASET_MAP 等，把字符串 env 名映射到内部使用的类型。  
- `WORKSPACE`、`Dockerfile` 等：Bazel / Docker 配置，与算法理解关系不大，可按需略读。
